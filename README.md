# ヘルパゴス Web版（MVP）

無人島サバイバル「ヘルパゴス（Hellapagos）」のオンライン対戦Web版。
ニセ協力型のコアループ「行動 → 生存判定 → 投票（生贄）→ 脱出」を、AIボット込みでオンラインで遊べる最小実装です。

設計の全体像は [DESIGN.md](DESIGN.md) を参照。

## 構成（npm workspaces モノレポ）

- `packages/shared` … 型定義・決定論的ゲームエンジン・AI（サーバ/クライアント共有）
- `packages/server` … Express + Socket.IO のサーバ権威ゲーム進行・AI駆動・状態リダクション
- `packages/client` … React + Vite のクライアント（ロビー / ゲーム盤 / 各フェイズUI）

## セットアップ

```bash
npm install
```

## 開発起動

```bash
npm run dev
```

- サーバ: http://localhost:8787
- クライアント: http://localhost:5173

ブラウザで `http://localhost:5173` を開き、ルームを作成 → 表示されたルームIDを別タブ/別端末で参加、またはAIボットを追加して開始します。

## 型チェック / テスト

```bash
npm run typecheck
npm test          # shared エンジンのユニットテスト
```

## 本番ビルド / デプロイ

単一オリジン構成（サーバがビルド済みクライアントも配信）です。

```bash
npm run build                      # packages/client/dist を生成
npm run start -w @hellapagos/server # サーバが dist を配信 → http://localhost:8787
```

Docker / Render / Fly.io での公開手順は [DEPLOY.md](DEPLOY.md) を参照（`Dockerfile` / `render.yaml` / `fly.toml` 同梱）。

## 遊び方

1. 毎日「天候 → 行動 → 生存判定 →（不足時）投票 → 脱出判定」を繰り返す。
2. 行動は **魚釣り / 水汲み / 木材集め / 難破船の探索** から1つ。
3. 生存者1人につき水1・食料1を毎日消費。足りなければ不足数だけ投票で追放。
4. 探索で得た資源は**自分の隠し財産**。生存判定で供出するか隠すか選べる。
5. 木材で**いかだの席**を作り、嵐が来るまでに席＋航海用の水食料を揃えて脱出すれば勝ち。

### 特殊アイテム（探索で入手）

| アイテム | 効果 |
|---|---|
| 🔫 拳銃 | 生存者1人を即座に射殺し、所持品を奪う（1回） |
| 💊 睡眠薬 | その追放投票で自分を対象外にする（1回） |
| 💉 解毒剤 | ヘビの毒（病気）を治す（1回） |
| 🪆 ブードゥー人形 | 死者1人を蘇らせる（1回・行動フェイズ） |
| 🪓 斧 / 🎣 釣り竿 / 🧪 浄水器 | それぞれ木材／魚／水の収量+1（永続） |

### AIの性格（4種・進行中は秘匿、終了時に公開）

| 性格 | 振る舞い |
|---|---|
| 🤝 協力的 | 不足を素直に供出する |
| 🗄️ 溜め込み屋 | 備蓄を抱え込みあまり出さない |
| 🎯 狙撃手 | 拳銃で裕福な相手を撃ち略奪する |
| 🐤 臆病者 | 疑われると睡眠薬で自衛し、群れに同調する |

投票フェイズでは性格に応じた煽りセリフをチャットに流します。

### その他の機能
- **ひとりで今すぐ**: トップの「▶ ひとりで今すぐ遊ぶ」でルーム作成→CPU補充→即開始を1クリック。
- **もう1戦**: 決着後に「🔁 同じ顔ぶれでもう1戦」で構成を保ったまま再戦。
- **指名贈与**: 資源カードなどを特定の相手へ「🎁 渡す」（貸し借り・取引・恩売り＝ニセ協力の駆け引き）。
- **出し渋りの可視化**: 生存ウィンドウで供出した人/しなかった人をバッジで表示。CPUは出し渋りを優先的に追放する。
- **抜け駆け脱出**: 全員ぶんの席が無くても出航を選べる。乗れた者だけ脱出、残席は島に残る（使った席は去る）。
- **モード**: ロビーで **ソロサバイバル**（単独勝利）／**短期決戦**（ハリケーンが早く来る）を切替。
- **実績バッジ**: 初脱出・単独生存・連続脱出などをローカルに記録し、トップ画面に表示。
- **議論チャット**: 投票前のなすり付け合いに。観戦者も発言可。CPUは（APIキーがあれば）Claudeで会話・交渉する。
- **観戦**: 進行中・満員のルームに参加すると観戦者として閲覧できる。
- **再接続（認証付き）**: リロードしても席へ自動復帰。サーバ発行の秘密トークンで保護され、他人のIDだけでは乗っ取れない。切断中はAIが代行。
- **答え合わせ＆戦績**: 終了時に各AIの性格を公開。通算戦績（脱出率・勝利数）をトップ画面に表示。
- **効果音・演出**: Web Audio 合成の効果音（🔊/🔇 で切替）、日付スプラッシュやイベントバナーなどのアニメ。`prefers-reduced-motion` に配慮。
- **モバイル/A11y対応**: レスポンシブ。狭い画面では「記録／チャット」をタブで切替。フェイズ遷移は aria-live で読み上げ。
- **ランキング**: 名前単位の通算成績をサーバが集計（`GET /leaderboard`）。トップ画面の「🏆 ランキング」から閲覧。
- **遊び方**: トップ画面の「📖 遊び方」でルールをいつでも確認できる。

### 開発・運用ツール
- **バランス計測**: `node --import tsx tools/sim.ts [卓数] [人数]` で全員CPUのモンテカルロ統計（早期投票率・平均終了R・脱出率など）。
- **CI**: `.github/workflows/ci.yml` が push/PR で typecheck・test（39件）・build を検証。
- **本番安全策**: 本番CORSは既定で同一オリジンのみ、LLMコストは同時実行/毎分上限、ルームGC、グレースフルシャットダウン（戦績フラッシュ）。詳細は [DEPLOY.md](DEPLOY.md)。

## E2E スモークテスト

サーバ起動中に `node smoke.mjs` を実行すると、ルーム作成→AI対戦→観戦参加→チャット往復→決着までを自動検証します。
