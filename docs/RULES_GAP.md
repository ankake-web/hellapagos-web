# RULES_GAP — 現行実装と公式ルールの差分（Phase 0）

現行は突貫MVP（`packages/shared/src/{engine,types,content,view}.ts` 計 ~1000行）。公式ルール仕様（依頼書）と突き合わせた差分を列挙する。★=要再実装の大物。

## 1. 乱数の核：木玉の袋 ★
- 現行: 釣り=一様乱数 `rng.int(1,3)`、木集め=固定 `+2(+斧)`。ヘビ（黒玉）は探索アイテム抽選には無く、実際の引きに**存在しない**。
- 公式: 袋に6玉（白5=魚 `1,1,2,2,3` / 黒1=ヘビ）。釣り・木集め両方がこの袋を引く（毎回戻す＝常に6玉）。
- 対応: 共有の `bag.ts` を新設。`drawBall(rng)` を釣り/木プッシュ両方で使用。

## 2. 筏トラック（0→6ループ）★
- 現行: `wood` を素のカウント、`raftCapacity = floor(wood / raftWoodPerSeat=2)`。
- 公式: 木ディスク 0→6 を1周するごとに筏カード1枚（=1座席）完成、ディスクは0へ。最大12座席。
- 対応: `raftSeats`（完成座席）＋ `raftProgress`（0..5）に置換。

## 3. 木集めのプッシュ ★
- 現行: 無し（固定+2）。
- 公式: 無条件+1 → 望めば追加1〜5を宣言し一度に引く。黒玉が無ければその数を加算、黒玉が出たら**病気**＋追加木は無し（最初の+1は残る）。
- 対応: 行動に `woodPush` 量を持たせる（人間はUI、CPUはリスク許容度で決定）。

## 4. 漂着物カード（54枚デッキ）★
- 現行: 探索で「アイテム7種の重み抽選」or「資源を数量加算」。`hand` は `{food,water}` の数量、`items` は7種。
- 公式: 54枚デッキ。配布=3〜8人各4枚/9〜12人各3枚、残りは山札。探索で山札トップ1枚を手札へ。資源も**カード**。
- 対応: `WreckageCard[]` の手札・デッキへ全面置換。下記カード一式を実装。

## 5. 天候デッキ（12枚＋ハリケーン下半分）★
- 現行: `stormIn` カウントダウン＋ rain/sunny ランダム。
- 公式: 12枚（各 降水量0〜3、うちハリケーン1枚）。設定で「ハリケーン+ランダム5＝6枚」をシャッフルして残り6枚の**下**に重ねる→ハリケーンは7〜12ターン目に出る。各ラウンド先頭で1枚公開し、その日の水汲み量(0〜3)が確定。
- 対応: `weatherDeck` を構築。`precip` で水汲み量を決定。0の日は汲めない。

## 6. 生存トラックの人数別初期値 ★
- 現行: 食料・水とも0スタート。
- 公式: 3人=5/6, 4=7/8, 5=8/10, 6=10/12, 7=12/14, 8=13/16, 9=15/18, 10=16/20, 11=18/22, 12=20/24。上限各36。
- 対応: 初期化テーブル＋ `clamp(0,36)`。

## 7. 生存チェックの順序と投票 ★
- 現行: `shortage = need - min(food,water)`、**同時**多人数追放。
- 公式: **水→食料の順**に各1消費。各資源で不足したら、不足分だけ**1人ずつ順次投票**で脱落（該当資源カードを出せば救済、銃で人数削減も可）。開始時に既に0ならその資源は投票なしで「カードを出せた者だけ生存」。
- 対応: 水・食料を別々に解決。順次単独投票へ。

## 8. 投票方式 ★
- 現行: 全員同時に最多得票者を、必要数まとめて追放。`voteImmune`(睡眠薬)で並べ替え。
- 公式: 親が合図→全員同時に1人を指す→最多が脱落。同数は**親が裁定**（親が当事者でも親が選ぶ）。1投票=1人。複数必要なら別投票。水晶玉=最後に投票。
- 対応: デジタルでは順次入力→確定→集計→開示（同時票と等価）。親裁定・水晶玉を実装。

## 9. 病気の仕様 ★
- 現行: `sick` で行動スキップ、`beginDay` で回復。
- 公式: 病気の間はそのラウンド終了時の**投票に参加不可**（脱落対象にはなる／脱落指定されたら水食料カードで自分は救える）。**次ラウンド**は行動・カード使用不可。噛まれたターンに血清は使える。
- 対応: `sick`（今ターン）と回復タイミングを正確化。投票参加可否を区別。

## 10. 行動の手番 ★
- 現行: 全員**同時提出**＋ readiness。
- 公式: 親から時計回りに1人1アクション。
- 対応: **ターン制**（`currentActorIndex`）へ。Phase2のCPU逐次演出とも整合。

## 11. カードはアクション外でいつでも使用
- 現行: アイテムは特定フェイズで割り込み可（近い）。
- 公式: 自分・仲間のターン中いつでも。デジタルでは「自分の手番＋生存/投票ウィンドウ」で近似（RULES.md に明記）。

## 12. カード一式（資源/単発/永続/無用品）★
- 現行: gun/pills/antidote/voodoo/axe/rod/filter の7種のみ。資源はカードでない。
- 公式: 水ボトル・汚れた水・サンドイッチ・イワシ缶・腐った魚・フルーツバスケット／血清・ブードゥー・睡眠薬・目覚まし時計／水筒・釣り竿・斧・水晶玉・銃・弾／無用品。
- 対応: データ駆動でカード効果を実装。

## 13. 脱落者の手札分配・銃回収 ★
- 現行: 脱落者の `hand` はプールへ、`items` は破棄。
- 公式: 脱落者の手札は**両隣に交互**配布。銃は回収（撃った側は被害者手札を取得）。
- 対応: 分配ロジックを実装。

## 14. 脱出条件 ★
- 現行: `capacity>=need && food>=need && water>=need`（席=floor(wood/2)）。
- 公式: 消費後に **筏座席≧生存者数** かつ **生存者×(水1・食料1)** が残れば全員乗船で勝ち。フルーツバスケットは旅の補給に使えない。ハリケーン公開ラウンドは終了時に必ず脱出（不足なら犠牲を出す）。
- 対応: 座席=`raftSeats`、補給=生存者×1。

## 15. 親の銃所持競合・弾
- 公式: 複数が銃所持なら最初に宣言した者が先に撃つ。弾は使用後失う。
- 対応: 宣言順・弾消費を実装。

## 改修方針（確定）
1. **エンジンを公式仕様で全面再実装**（袋・筏トラック・天候デッキ・生存表・水→食料順・順次単独投票・病気・54枚カード・脱出/ハリケーン）。`docs/RULES.md` を正典とし一致させる。
2. 行動フェイズを**ターン制**へ。サーバ/クライアント/ビューを新モデルへ追従。
3. CPUは async 逐次・性格/難易度・木プッシュのリスク判断・投票AI（Phase 2）。
4. 演出は CSS/SVG/絵文字/Web Audio で実装（Phase 3）。
5. RULES 準拠の自動テスト（Phase 4）→ 統合通しプレイ（Phase 5）。

判断を要した既定値（RULES.md に正式記載）:
- 魚分布 `1,1,2,2,3`＋黒（依頼書の妥当値）。
- 天候デッキの降水量分布、54枚カードの内訳枚数（公式が枚数非公開のため妥当配分）。
- 「カードいつでも使用」はデジタルでは手番＋生存/投票ウィンドウに限定。
